next permutation

http://www.lintcode.com/en/problem/next-permutation/
I know how to solve this problem for a long time, but still I made some mistakes while I am implementing it.
The algorithm is pretty straight forward. One first need to find the maximum value k that nums[k] < nums[p] where k < p. The second step is to find the minimum value for all p > k. If duplicate is found, then find the last one. Third step,
swap nums[k] and nums[p], after that reverse all numbers on the right side of k.
The first error is in line 36, I forgot to incrementing and decrementing j and k.
The second error is that I did not think carefully about duplicate on line 24.
The third problem is that I need to swap all numbers on the right side of k, still on line 36.
